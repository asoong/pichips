PiChip Viewer — Architecture

How synthetic-data training, the Raspberry Pi, and your Mac all connect to count poker chips in real time.

Web client (cloud) Your Mac Raspberry Pi Where you watch

What happens at each stage

  1. 1Web client — generate synthetic chip images, segment + label them into 9 classes (<color>_<face|edge>), curate, and export a YOLO-format dataset.zip.
  2. 2Mac — train.py fine-tunes a YOLO model on that dataset (Apple-Silicon MPS) and writes models/pichip_detector.pt. The class names are baked into the weights.
  3. 3Pi — deploy_model.sh exports the model to ONNX (~1.7× faster on the Pi CPU). overlay_viewer.py grabs camera frames via picamera2, runs detection in a background thread (so the feed stays ~15 fps while detections refresh asynchronously), derives color + orientation from the class names, edge-counts side-lying stacks, and draws the value HUD.
  4. 4Watch it — the annotated frames are served as an MJPEG HTTP stream on port 8090 (open in a browser on your Mac) and/or shown in a window on a monitor plugged into the Pi.

Commands

# 1) Train (Mac), after exporting a dataset from the web client
python train.py --dataset datasets/<token> --model yolo11n.pt \
  --epochs 100 --imgsz 1024 --batch 16 --device mps

# 2) Deploy to the Pi
scp models/pichip_detector.pt <pi-user>@pichip.local:~/pichip_viewer/models/
ssh <pi-user>@pichip.local 'cd ~/pichip_viewer && ./deploy_model.sh'

# 3) Run on the Pi (headless over SSH, or a window on an attached monitor)
ssh <pi-user>@pichip.local 'cd ~/pichip_viewer && ./run.sh'

# 4) Watch from your Mac — open in a browser:
#    http://pichip.local:8090/

Full setup, env vars, and performance notes are in README.md; the end-to-end runbook (incl. how many training images to use) is in TRAINING.md.
